HeadHunter Parser

Консольное приложение для получения, хранения и анализа вакансий с сайта hh.ru

Программа взаимодействует с официальным API HeadHunter, сохраняет найденные вакансии в формате JSON, позволяет просматривать сохранённые данные, выводить топ по зарплатам и удалять ненужные записи.

Проект реализован с соблюдением принципов модульности и покрыт тестами.
Все операции выполняются через простое консольное меню.

Основной функционал

Поиск вакансий по ключевому слову через API HeadHunter

Сохранение найденных вакансий в файл hh_vacancies.json

Просмотр всех сохранённых вакансий (vacancies.json)

Вывод топа N вакансий по зарплате

Удаление вакансий по названию

Простое интерактивное управление через консоль

Описание модулей
main.py

Главный сценарий программы, обеспечивающий интерактивное взаимодействие с пользователем.
Реализует консольное меню и объединяет функциональность всех остальных модулей.

Основные возможности:

Загружает вакансии по ключевому слову с API hh.ru и сохраняет их в hh_vacancies.json.

Показывает сохранённые вакансии из vacancies.json.

Отображает топ N вакансий по уровню зарплаты.

Удаляет вакансию по названию (title).

Завершает работу программы.

api.py

Отвечает за интеграцию с API HeadHunter и загрузку вакансий.

Содержит:

Parser — абстрактный класс, определяющий интерфейс для работы с внешними API.
Задаёт обязательные методы load_vacancies() и get_vacancies().

HeadHunterAPI — реализация класса Parser, взаимодействующая с API hh.ru.

Основные методы:

load_vacancies(keyword) — отправляет GET-запрос к https://api.hh.ru/vacancies, получает вакансии по ключевому слову и сохраняет их во внутренний список.

get_vacancies() — возвращает загруженные данные в виде списка Python-объектов (list[dict]).

models.py

Содержит модель сущности вакансии — класс Vacancy.

Атрибуты:

title — название вакансии

url — ссылка на вакансию

salary — зарплата (число или словарь {from, to})

description — краткое описание

Основные методы и свойства:

salary — вычисляет числовое значение зарплаты; если задан диапазон {from, to}, возвращает среднее значение.

__eq__ и __lt__ — реализуют сравнение вакансий по зарплате.

to_dict() — преобразует объект Vacancy в словарь для последующей записи в JSON.

from_api_to_dict() — создаёт объект Vacancy из данных, полученных через API.

storage.py

Отвечает за сохранение, чтение и удаление вакансий из файлового хранилища (JSON).

Содержит:

VacancyStorage — абстрактный класс, задающий интерфейс для хранилища вакансий.
Обязательные методы:
add_vacancy(), get_all_vacancies(), delete_vacancy().

JSONStorage — конкретная реализация VacancyStorage, использующая JSON-файлы.

Основные особенности реализации:

При инициализации создаёт файл, если он отсутствует.

add_vacancy() — добавляет новую вакансию в JSON.
Проверяет дубликаты по url.
При совпадении названия добавляет индекс (Python Developer (1) и т. д.).

get_all_vacancies() — читает JSON и возвращает список объектов Vacancy.

delete_vacancy() — удаляет вакансию по названию (title).
